# 17.2 Перевірка типу за допомогою оператора is

Конкретний тип об'єкта можна визначити за допомогою оператора is . Нижче наведена його загальна форма:

вираз is тип

Де вираз означає окремий вираз, що описує об'єкт, типякого перевіряється. Якщовиразмає сумісний або такий самий тип, як і перевіряється тип , то результат цієї операції стає істинним, в іншому випадку — хибним. Так, результат буде істинним, якщо вираз має перевіряємий тип у тій або іншій формі. У операторі isобидва типи визначаються як сумісні, якщо вони одного й того ж типу або якщо передбачено перетворення посилань, упакування або розпакування.

Лістинг 17.1 - Застосування оператора is

using System;
class A { }
class В : A { }
class UseIs
{
    static void Main()
    {
        A a = new A();
        В b = new В();
        if (a is A)
            Console.WriteLine("а має тип A");
        if (b is A)
            Console.WriteLine("b сумісний з А, оскільки він похідний від А");
        if (a is В)
            Console.WriteLine("Не виводиться, оскільки а не похідний від В");
        if (b is В)
            Console.WriteLine("В має тип В");
        if (a is object)
            Console.WriteLine("а має тип object");
    }
}

Ось до якого результату призводить виконання цього коду.

а має тип А b сумісний з А, оскільки він похідний від А b має тип В а має тип object

Більшість виразів is у наведеному вище прикладі не потребує пояснень, але два з них все ж потрібно роз'яснити. Попередньо, зверніть увагу на наступний рядок коду.
