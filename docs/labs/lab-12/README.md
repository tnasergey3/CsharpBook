# Система контролю версіями Git. Платформа GitHub

[◀️ До змісту лабораторних](README.md)

---

Мета роботи: розглянути: сервіс контролю версій на базі GIT під назвою GitHub.

Обладнання: технічне забезпечення: персональний комп’ютер, або ноутбук; програмне забезпечення: terminal, power shell, GitHub.

Порядок виконання роботи

Ознайомитися з теоретичними відомостями та виконати поставлені задання:

Опрацювати 11-29 розділи ресурсу https://githowto.com/uk та додати результати виконання до звіту.

Створити обліковий запис в сервісі GitHub https://github.com/

Встановити GitHub Desktop https://desktop.github.com/

Використайте на прикладах GitHub Desktop.

Створити публічний репозиторій, виконати операцію клонування репозиторію.

Додати в репозиторій декілька виконаних програм із лабораторних робіт.

Створити гілку із назвою fix-issues та змінити код програми після чого залити гілку на сервер GitHub.

Додати до свого репозиторію декілька своїх одногрупників або викладача як contributors.

Контрольні питання

Що таке GitHub.

Розкрити поняття гілки.

Розкрити поняття злиття гілок.

Розкрити поняття перенесення (rebase).

Які є способи вирішення конфліктів.

Які найкращі практики для іменування гілок.

Як переключитися з гілки на гілку.

Література

Галуження в Git – Гілки у кількох словах

GitHub – Створення та налаштування облікового запису.
