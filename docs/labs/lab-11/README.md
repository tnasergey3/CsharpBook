# Введення в системи контролю версій

[◀️ До змісту лабораторних](README.md)

---

Мета роботи: розглянути: системи контролю версій, ознайомитися із системою контролю версій GIT, освоїти базові поняття та команди.

Обладнання: технічне забезпечення: персональний комп’ютер, або ноутбук; програмне забезпечення: terminal, power shell.

Порядок виконання роботи

Ознайомитися з теоретичними відомостями та виконати поставлені задання:

Опрацювати 1-10 розділи ресурсу https://githowto.com/uk та додати результати (скріншоти) виконання до звіту.

Встановити GIT на свій комп’ютер та викликати команду git --version.

Провести початкове налаштування GIT використовуючи команду git config. Створити репозиторій використовуючи команду git init.

Додати декілька текстових файлів використовуючи команду git add file.txt та створити декілька комітів використовуючи команду git commit -m “message”.

Перевірити статус файлів використовуючи команду git status.

За допомогою команди git log перевірити історію створених комітів.

Додати зміни в текстовий файл який ви додали та зафіксували в коміт після чого зробити новий коміт зі зміненим файлом, далі виконати команду git log щоб отримати хеш попереднього коміту та використовуючи команду git checkout <hash> повернути версію файла до ваших змін.
